# Bee — Agent Guide

## Critical Rules

1. **Always use `direnv` / `.envrc` for environment variables.** Before running `cargo build --release` or `xcodebuild`, run `direnv allow` or `eval "$(direnv export bash)"` to load `.envrc`. The `.envrc` sets `MLX_SYS_PREFIX`, `HF_TOKEN`, and other required env vars. Do NOT try to set them manually.

2. **Do not edit generated files.** Files marked `@generated` (e.g., `swift/vox-shared/AppIPC.swift`, `swift/vox-shared/ImeIPC.swift`) are generated by `cargo run --bin bee-rpc-codegen`. The C header `rust/bee-ffi/include/bee_ffi.h` is generated by cbindgen via `rust/bee-ffi/build.rs`. Edit the source, then regenerate.

3. **Xcode project is generated by xcodegen.** Do not edit `bee.xcodeproj` directly. Edit `swift/bee-project.yml` then run `xcodegen generate --spec bee-project.yml` from the `swift/` directory.

4. **Use facet/facet-json, not serde.** All types in bee-types, bee-phonetic, and bee-correct use `#[derive(Facet)]` for serialization. Never add serde derives.

5. **cbindgen for Rust/Swift FFI.** The C header is auto-generated. Add `#[no_mangle] pub unsafe extern "C" fn` functions to `rust/bee-ffi/src/lib.rs` and they appear in the header after `cargo build -p bee-ffi`.

## Project Structure

```

├── swift/                           # macOS app + IME (Xcode)
│   ├── bee-project.yml              # xcodegen spec (source of truth)
│   ├── bee/                         # Main app target

│   │   └── BeeVoxIMEClient.swift

```

## Key Commands

```bash
# Load environment (ALWAYS do this first)
direnv allow  # or: eval "$(direnv export bash)"

# Build Rust FFI library
cargo build --release -p bee-ffi

# Regenerate IPC Swift code (MANDATORY after ANY change to rust/bee-rpc/src/lib.rs)
# This generates swift/vox-shared/{AppIPC,ImeIPC,BeeIPC}.swift directly from the Rust service traits.
cargo run --bin bee-rpc-codegen

# Regenerate Xcode project (after changing bee-project.yml)
cd swift && xcodegen generate --spec bee-project.yml

# Build macOS app
cd swift && xcodebuild -project bee.xcodeproj -scheme bee -configuration Release build

# Run eval harness
cargo run -p beeml -- eval

# Run tests
cargo test -p bee-correct
cargo test -p bee-phonetic
```

## Dependencies

- **vox**: RPC framework (Swift package, pinned by revision in bee-project.yml)
- **MLX**: Apple ML framework (requires prebuilt artifacts at `$MLX_SYS_PREFIX`)
- **espeak-ng**: Bundled for G2P in bee-correct
