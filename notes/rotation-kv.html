Rotation + Persistent Decoder KV

This document defines a coherent streaming model for persistent decoder KV in Bee. It also calls out the invalid model explicitly: persistent KV without audio cuts.

Core Claim

State Vectors

committed / finalized
retained stable context
revisable rollback tail
decoder KV-visible positions

Mechanism (Ordered)

This is the precise order for each commit/rotation cycle when persistent KV is enabled.

  • 1. Decode chunks with persistent KV, but keep rollback window N revisable.
  • 2. Build full current transcript (excluding metadata) and run ZIPA alignment on full current audio buffer.
  • 3. Choose textual commit partition [C][P][R] (commit/carry/rollback), then snap to preferred boundary.
  • 4. From alignment, derive commit_end_time for the end of C.
  • 5. Cut audio at commit_end_time: audio = committed_audio + remaining_audio.
  • 6. Compute exact token index mapping for kept text (P + R) and exact decoder position for that same kept prefix.
  • 7. Truncate/rebase decoder KV to that exact position (or rebuild KV from kept tokens if truncation-only addressing is insufficient).
  • 8. Start next sub-session from remaining audio + kept text/KV state. Never re-feed committed audio.

Why It Works

  • ZIPA provides a time-grounded boundary, so audio trimming is physically meaningful.
  • Token ownership over aligned phones lets that same boundary map back to text/token indices.
  • Decoder KV truncation/rebase at the same boundary preserves decoder history consistency.
  • All frontiers move together; no duplicated committed history is reintroduced.

Failure Modes

  • If token-boundary mapping is approximate (not exact), KV can desynchronize from text and drift.
  • If audio is not cut at the aligned boundary, persistent KV becomes invalid for streaming.
  • If boundary snapping changes text partition after timing is chosen, boundary must be recomputed from the snapped plan.
  • If we cannot express KV as suffix-retained state after cut, we must rebuild KV from kept tokens explicitly.

Verdict

Persistent decoder KV can work in Bee, but only in a synchronized rotation model: ZIPA cut time -> audio trim -> exact token boundary -> KV truncate/rebase. Persistent KV without aligned audio cutting is not a valid streaming design.
